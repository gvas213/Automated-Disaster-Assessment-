# v5: Diverse-Input Voting

## Problem with v4

v4 got 72.9% accuracy (big jump from v3) but still has 157 false positives on no-damage buildings. Root causes:

1. **Histogram equalization amplifies noise** — the diff image normalizes to full brightness range, so even tiny compression/alignment artifacts look like bright "change." A nearly-blank diff gets equalized to look dramatic.
2. **Upscaling before diffing creates artifacts** — Lanczos interpolation on pre vs post produces slightly different sub-pixel values, creating fake differences in the diff image.
3. **Same inputs × 3 votes = same wrong answer × 3** — voting adds no value when all voters see identical data. The "ensemble" is an illusion.

## v5 Core Idea: Each Vote Gets Different Inputs

Instead of 3 identical calls, each vote sees a genuinely different representation of the same building:

### Vote Diversity Options

**Different diff methods per vote:**
- Vote 1: Grayscale diff + histogram equalization (current)
- Vote 2: RGB color diff (no grayscale conversion) — preserves color-based damage signals
- Vote 3: Raw grayscale diff WITHOUT equalization — avoids noise amplification

**Different crop scales per vote:**
- Vote 1: Original padding (150px)
- Vote 2: Tighter crop (75px) — forces focus on the building itself
- Vote 3: Wider crop (250px) — more surrounding context for comparison

**Different upscale factors per vote:**
- Vote 1: 2x upscale
- Vote 2: No upscale (raw resolution)
- Vote 3: 3x upscale

**Different prompt framings per vote:**
- Vote 1: Standard CoT classify prompt
- Vote 2: Conservative prompt ("assume no damage unless you see clear structural change")
- Vote 3: Sensitive prompt ("flag any potential change, even subtle ones")

### Disagreement = Signal

When all 3 votes agree → high confidence in the classification.
When votes disagree → the building is ambiguous, default toward less damage (conservative tiebreak).

## Cropping API

`crop_buildings()` in `cropping.py` now accepts an optional `padding` parameter (default 150, backward compatible):

```python
crop_buildings(pre_img, post_img, post_json, padding=75)   # tight crop
crop_buildings(pre_img, post_img, post_json, padding=150)  # default
crop_buildings(pre_img, post_img, post_json, padding=250)  # wide crop
```

This enables the different-crop-scales-per-vote strategy without breaking any existing code.

## No Gate

The binary gate is removed in v5. It fundamentally doesn't work:
- Too tight → blocks real damage (v3: 0% recall on damaged buildings)
- Too loose → passes everything (v3 fixed: 527 false positives)
- No sweet spot exists for a binary decision on a spectrum problem

v4 proved gateless is better — removing the gate and classifying all 4 levels in a single pass jumped accuracy from 16-27% to 72.9%. The diff image + baseline description + CoT naturally calibrates without a gate.

v5 stays gateless like v4, focuses on diverse inputs per vote instead.
